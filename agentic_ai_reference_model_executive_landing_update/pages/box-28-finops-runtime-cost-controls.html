Constrain token, API, model and workflow consumption during execution.

This capability enforces runtime budgets, quotas, loop limits, rate limits and anomaly detection to prevent agents from creating economic denial-of-service or uncontrolled consumption.

Purpose and strategic objective

Purpose

To keep AI runtime usage within approved economic and operational boundaries.

Strategic objective

Enable AI scale without uncontrolled token, infrastructure, model-provider or API spend.

Architectural placement

Why this sits here

This sits in Layer 2 because cost risk materialises while agents operate, retry, call APIs, loop or fan out into tools and child agents.

Cost is a runtime safety boundary, not only a finance report.
Problem statement

Agents can enter loops, overuse expensive models, call high-cost APIs or spawn parallel tasks faster than traditional budget controls can detect.

Capability description

What the capability does

  • Enforces token, API and workflow budgets.
  • Applies rate limits, loop limits and retry caps.
  • Detects cost anomalies by agent, owner and lineage.
  • Throttles, pauses or terminates runaway workflows.
  • Feeds economic metrics to Layer -4.

What the capability does not do

  • It does not determine business value by itself.
  • It does not replace portfolio economics.
  • It does not guarantee cost efficiency without design optimisation.
