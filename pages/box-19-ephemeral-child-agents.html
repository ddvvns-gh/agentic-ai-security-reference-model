Treat child agents as short-lived, scoped non-human identities.

This capability governs dynamically spawned child agents as short-lived non-human identities. Each child agent must have scoped purpose, cryptographic identity, parent lineage, restricted token exchange and automatic expiry.

Purpose and strategic objective

Purpose

To prevent temporary child agents from becoming invisible, overprivileged or persistent attack surfaces.

Strategic objective

Enable dynamic multi-agent execution while maintaining identity, lineage, expiry and least privilege.

Architectural placement

Why this sits here

This sits in Layer 0 because child agents are active agent actors in the attack surface, even if they are temporary.

Temporary does not mean unmanaged; ephemeral agents still require identity and evidence.
Problem statement

Agents may spawn sub-agents for parallel work. If these child agents inherit broad user or parent privileges, the enterprise loses control of delegated authority.

Capability description

What the capability does

  • Registers child agents automatically at spawn time.
  • Issues short-lived, task-specific identities.
  • Limits delegated authority to specific sub-task, data and tool scope.
  • Expires and revokes child-agent credentials after completion.
  • Captures parent-child intent and lineage evidence.

What the capability does not do

  • It does not allow child agents to inherit cumulative user permissions.
  • It does not permit unbounded dynamic spawning.
  • It does not replace sandboxing or tool-gateway enforcement.

v18 hardening — Semantic Blast-Radius Isolation

Agent-to-agent communication is now treated as a semantic trust boundary. Downstream agents must not pass unvalidated raw natural-language blocks directly into a parent orchestrator or peer agent context. Outputs crossing agent boundaries require typed schemas, field-level validation, provenance tagging, classification and quarantine rules for ambiguous content.

Outcomes delivered

Dynamic control

Child agents can operate without becoming unmanaged.

Least privilege

Delegated authority is bounded to the task.

Lifecycle hygiene

Temporary agents expire automatically.

Enterprise examples

Research swarm

Parent creates child agents to summarise documents.

Normal / desired state:
Each child has read-only, task-bound identity and expiry.
Risk / control concern:
Child agents inherit broad parent access.

Parallel validation

Parent creates validators for separate records.

Normal / desired state:
Each validator can access only assigned record set.
Risk / control concern:
Validators can query unrelated data.
Failure scenarios

Runaway spawn

Agent creates many child agents without control.

Privilege inheritance failure

Child receives full parent permissions.

No expiry

Child agent continues after workflow ends.
