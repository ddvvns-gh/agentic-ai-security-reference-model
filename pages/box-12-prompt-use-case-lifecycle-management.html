Treat prompts, model settings and agent use cases as governed production artefacts.

This capability ensures use cases, prompts, system instructions, model configuration and agent logic are versioned, reviewed, approved and promoted through controlled lifecycle processes.

Purpose and strategic objective

Purpose

To prevent uncontrolled changes to AI behaviour through unmanaged prompts, model settings or use-case expansion.

Strategic objective

Apply SDLC discipline to AI-specific artefacts so behaviour changes are visible, approved, testable and reversible.

Architectural placement

Why this sits here

It sits in Layer -1 because it governs AI artefacts before production deployment and after the Prodspec is defined.

A prompt change can be a production behaviour change and should be governed like code.
Problem statement

Many AI behaviours are controlled through prompts, model parameters, retrieved context and tool instructions. If these are changed outside lifecycle control, production behaviour can change without security, risk or business review.

Capability description

What the capability does

  • Versions prompts, system instructions and model settings.
  • Routes use cases through risk-tiered approval.
  • Supports peer review and change history.
  • Controls promotion across environments.
  • Links prompt changes to testing and release gates.

What the capability does not do

  • It does not replace runtime inspection.
  • It does not guarantee prompt quality without testing.
  • It does not replace business ownership.

Outcomes delivered

Change control

AI behaviour changes are reviewed before release.

Traceability

Outputs can be linked to prompt and configuration versions.

Governed expansion

Agents cannot silently expand into new use cases.

Enterprise examples

System prompt update

A team updates a customer-service agent prompt.

Normal / desired state:
The prompt change is reviewed, tested and promoted.
Risk / control concern:
A quick prompt edit changes refund behaviour without review.

New use case

A reporting agent is repurposed for customer advice.

Normal / desired state:
The new use case is risk assessed and approved.
Risk / control concern:
The agent is used for regulated advice without oversight.
Failure scenarios

Hotfix prompt drift

A production prompt is edited directly.

Unknown prompt version

Incident investigators cannot identify which prompt produced an unsafe output.

Use-case creep

An agent gradually expands into high-risk decisions.
