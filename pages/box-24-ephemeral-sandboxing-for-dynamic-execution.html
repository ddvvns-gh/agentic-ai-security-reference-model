Route dynamic execution into isolated, disposable environments.

This capability ensures generated scripts, analysis code, transformations and experimental actions run in disposable sandboxes with strict network, filesystem, identity and resource limits.

Purpose and strategic objective

Purpose

To allow useful dynamic execution without exposing trusted hosts, credentials or data stores.

Strategic objective

Reduce risk from generated code by executing it in external, deterministic isolation rather than relying on the agent to behave safely.

Architectural placement

Why this sits here

This sits in Layer 1 because sandboxing is a runtime foundation for safe dynamic execution.

Generated code should be treated as untrusted until isolated, constrained and observed.
Problem statement

Agents often generate or execute code. Without sandboxing, prompt injection or model error can become host compromise, data exfiltration or persistent malware.

Capability description

What the capability does

  • Routes dynamic execution into approved sandboxes.
  • Applies network, filesystem, resource and credential constraints.
  • Uses disposable containers, microVMs or Wasm isolation where appropriate.
  • Captures execution evidence and output artefacts.
  • Destroys sandbox state after task completion.

What the capability does not do

  • It does not make all code safe.
  • It does not replace code review for production changes.
  • It does not allow bypass of data or tool policy.

Outcomes delivered

Safe computation

Agents can compute without host exposure.

Blast-radius reduction

Unsafe code is isolated and time-bound.

Execution evidence

Dynamic behaviour is logged for review.

Enterprise examples

Data analysis script

Agent generates Python to analyse data.

Normal / desired state:
Script runs in sandbox with approved data mount.
Risk / control concern:
Script runs on production host with network access.

Generated remediation command

Agent proposes system fix.

Normal / desired state:
Command is tested in sandbox first.
Risk / control concern:
Command executes directly in production.
Failure scenarios

Host compromise

Generated code executes on trusted host.

Open egress

Sandbox has unrestricted outbound network.

Persistent artefacts

Sandbox state remains after task completion.
