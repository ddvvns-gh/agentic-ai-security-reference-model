Isolate dynamic code execution required by agents.

This capability routes dynamic scripts, code generation, data transformations and computational tasks into isolated, time-bound sandboxes with restricted network, filesystem and resource access.

Purpose and strategic objective

Purpose

To allow useful dynamic execution without exposing the host, data environment or broader enterprise systems.

Strategic objective

Enable safe experimentation and computation while constraining blast radius.

Architectural placement

Why this sits here

This sits in Layer 1 because sandboxing is a trusted execution pattern supporting dynamic agent workloads.

Agents may need to execute code, but they should not execute it directly on trusted hosts.
Problem statement

Some agents need to run generated code, transform data or test scripts. Without sandboxing, dynamic execution can compromise hosts, exfiltrate data or persist malicious artefacts.

Capability description

What the capability does

  • Creates isolated execution environments.
  • Applies network, filesystem and resource limits.
  • Expires sandboxes after task completion.
  • Captures execution logs and outputs.
  • Supports malware and data-leak controls.

What the capability does not do

  • It does not make all generated code safe.
  • It does not replace review for production code.
  • It does not allow bypass of data or tool policies.

Outcomes delivered

Safe computation

Agents can compute without host exposure.

Blast-radius reduction

Unsafe code is isolated and time-bound.

Execution evidence

Dynamic behaviour is logged for review.

Enterprise examples

Data analysis script

Agent generates Python to analyse a dataset.

Normal / desired state:
Script runs in sandbox with approved data access.
Risk / control concern:
Script runs on production host with network access.

Generated remediation command

Agent proposes system fix.

Normal / desired state:
Command is tested in sandbox first.
Risk / control concern:
Command is executed directly on production.
Failure scenarios

Host compromise

Generated code executes on trusted host.

Data exfiltration

Sandbox has unrestricted outbound network.

Persistent artefacts

Sandbox state remains after task completion.
