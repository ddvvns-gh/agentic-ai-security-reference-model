Inspect prompts and outputs as one layer of runtime protection.

This capability analyses incoming prompts, retrieved context and generated outputs for injection attempts, sensitive data, unsafe instructions, policy violations and malicious content.

Purpose and strategic objective

Purpose

To reduce exposure to prompt injection, unsafe outputs and data leakage during live AI interactions.

Strategic objective

Provide runtime risk reduction while recognising semantic inspection is only one layer and cannot fully solve prompt injection.

Architectural placement

Why this sits here

This sits in Layer 2 because inspection happens while the agent is operating and processing prompts, context and outputs.

Inspection reduces risk, but must be combined with least privilege, tool controls and data boundaries.
Problem statement

AI agents process natural language and retrieved context that can carry malicious instructions or sensitive data. Traditional content controls are not sufficient for semantic attacks.

Capability description

What the capability does

  • Inspects direct prompts and retrieved context.
  • Checks outputs for sensitive data, unsafe content and policy issues.
  • Flags or blocks suspicious semantic patterns.
  • Feeds telemetry and incident response.
  • Supports tuning and false-positive review.

What the capability does not do

  • It does not fully prevent prompt injection.
  • It does not replace tool-level authorization.
  • It does not prove output correctness.

Enterprise examples

RAG document attack

A document contains hidden instructions.

Normal / desired state:
Retrieved context is inspected before reaching agent.
Risk / control concern:
Agent follows malicious document instructions.

Generated email

Agent drafts customer email.

Normal / desired state:
Output is checked for sensitive data and policy tone.
Risk / control concern:
Email reveals internal investigation details.
Failure scenarios

Indirect injection bypass

Poisoned context manipulates tool usage.

Sensitive output

Agent includes customer data in public response.

Malicious code output

Agent emits credential-stealing script.
