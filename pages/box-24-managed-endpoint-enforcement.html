Enforce endpoint and browser controls that reduce local AI risk.

This capability provides local enforcement on managed endpoints, including privilege management, browser controls, DLP, process restrictions and monitoring for AI-driven interactions and generated code execution.

Purpose and strategic objective

Purpose

To prevent endpoint-level AI misuse, data leakage and unsafe local execution on managed devices.

Strategic objective

Create a trusted execution baseline for employees interacting with AI services from enterprise-managed devices.

Architectural placement

Why this sits here

This sits in Layer 1 because it is part of the declarative execution baseline that supports safe AI use at the device layer.

Network AI controls are insufficient if the endpoint can leak data or execute unsafe generated code.
Problem statement

AI risk often materialises through browser actions, local files, clipboard transfers, downloads, extensions and script execution.

Capability description

What the capability does

  • Enforces privilege and execution policies.
  • Controls browser uploads, downloads, copy/paste and extensions.
  • Monitors local AI tool usage.
  • Blocks unauthorised local execution.
  • Feeds endpoint telemetry into detection.

What the capability does not do

  • It does not control unmanaged devices.
  • It does not replace AI runtime inspection.
  • It does not judge business correctness of outputs.

Enterprise examples

Developer laptop

Developer uses AI to generate shell commands.

Normal / desired state:
Commands require review or sandboxed execution.
Risk / control concern:
Generated command deletes or exfiltrates files.

Browser upload

User uploads data to an AI site.

Normal / desired state:
DLP blocks sensitive file upload.
Risk / control concern:
Sensitive files leave enterprise control.
Failure scenarios

EPM bypass

Generated code elevates privilege.

Clipboard leakage

User pastes credentials into AI assistant.

Unapproved local model

Local model stores sensitive prompts.
