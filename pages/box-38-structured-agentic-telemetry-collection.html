Collect AI-specific telemetry needed for detection, assurance and governance.

This capability collects structured telemetry across prompts, outputs, tool calls, memory, decisions, approvals, policies, cost and runtime signals. It creates the observability substrate for AI security operations.

Purpose and strategic objective

Purpose

To make AI behaviour observable in a way that traditional logs cannot achieve alone.

Strategic objective

Enable detection, incident response, control testing, audit and performance management across the AI workforce.

Architectural placement

Why this sits here

This sits in Layer 4 because it supports continuous operation, monitoring and assurance after deployment.

You cannot secure, govern or assure what you cannot observe.
Problem statement

Traditional application logs do not capture agent reasoning context, tool intent, prompt lineage, policy outcomes or output commitment state.

Capability description

What the capability does

  • Defines AI telemetry schema and event types.
  • Collects prompt, output, tool, memory and policy events.
  • Correlates agent, user, model, data and action identifiers.
  • Supports privacy-aware logging and minimisation.
  • Feeds SIEM, XDR, GRC and metrics.

What the capability does not do

  • It does not require storing sensitive chain-of-thought.
  • It does not replace decision evidence schema.
  • It does not automatically detect threats without analytics.

Enterprise examples

Tool call monitoring

Agent invokes external API.

Normal / desired state:
Telemetry captures actor, purpose, tool and decision.
Risk / control concern:
Only API endpoint log exists without AI context.

Prompt inspection event

Potential injection is detected.

Normal / desired state:
Event links to session and downstream action.
Risk / control concern:
Alert cannot be correlated to agent outcome.
Failure scenarios

Missing telemetry

Production agent emits no AI-specific events.

Sensitive log exposure

Full prompts with personal data are over-retained.

No correlation ID

Incident team cannot connect prompt to tool call.
