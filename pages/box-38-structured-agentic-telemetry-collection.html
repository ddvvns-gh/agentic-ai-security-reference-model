Collect telemetry that supports identity chains, external enforcement and intent provenance.

This capability collects privacy-aware telemetry across identity, prompts, tool requests, policy decisions, sandbox execution, outputs, approvals and decision lineage. It is designed to support semantic reconstruction and external control assurance.

Purpose and strategic objective

Purpose

To make agent behaviour observable in a way that traditional logs cannot achieve.

Strategic objective

Enable detection, incident response, control testing, audit and assurance across agentic workflows.

Architectural placement

Why this sits here

This sits in Layer 4 because telemetry supports continuous operation, monitoring and assurance after deployment.

Telemetry must capture the control-plane story, not only infrastructure events.
Problem statement

Traditional logs miss the context that matters for AI: delegated authority, tool intent, retrieved context, policy routing and output commitment state.

Capability description

What the capability does

  • Defines AI telemetry events and correlation identifiers.
  • Captures identity chain, tool intent, policy decisions and enforcement outcomes.
  • Collects sandbox, gateway and output commitment telemetry.
  • Minimises or masks sensitive content where appropriate.
  • Feeds SIEM, XDR, GRC, metrics and evidence stores.

What the capability does not do

  • It does not require storing hidden chain-of-thought.
  • It does not replace decision lineage evidence.
  • It does not detect threats without analytics and response logic.

Enterprise examples

Tool gateway event

Agent requests API call.

Normal / desired state:
Telemetry captures agent identity, token, parameters, policy and outcome.
Risk / control concern:
Only API server log exists.

Sandbox execution

Agent runs generated code.

Normal / desired state:
Telemetry captures sandbox ID, resources, network and output.
Risk / control concern:
Execution is invisible after sandbox deletion.
Failure scenarios

Missing policy event

SOC sees action but not policy decision.

No session ID

Prompt cannot be linked to tool call.

Sensitive prompt logging

Full personal data is retained unnecessarily.
