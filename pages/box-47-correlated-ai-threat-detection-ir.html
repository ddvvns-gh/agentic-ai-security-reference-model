Detect attacks by correlating semantic, identity and deterministic control signals.

This capability correlates prompts, identity chains, tool gateway decisions, sandbox events, data movement, output commitments and decision lineage to detect AI-specific attacks and grey failures.

Purpose and strategic objective

Purpose

To provide security operations coverage for agentic AI threats that traditional SOC rules miss.

Strategic objective

Detect, contain and learn from prompt injection, tool abuse, identity misuse, data leakage, sandbox escape and self-policing failures.

Architectural placement

Why this sits here

This sits in Layer 4 because it operationalises monitoring and response after deployment.

AI incidents cross semantic, identity, runtime and business layers; detection must correlate all of them.
Problem statement

A prompt injection may look like a normal API call unless the SOC can connect prompt context, delegated identity, tool parameters and policy decisions.

Capability description

What the capability does

  • Defines AI-specific detection use cases.
  • Correlates semantic signals with identity, gateway, sandbox and output events.
  • Detects direct-path bypass and self-policing reliance.
  • Triggers containment, escalation or tuning playbooks.
  • Feeds lessons learned into controls and Prodspec updates.

What the capability does not do

  • It does not replace deterministic preventive controls.
  • It does not guarantee every semantic attack is detected.
  • It does not replace business-led incident ownership.

Enterprise examples

Indirect injection

Poisoned RAG context triggers suspicious tool call.

Normal / desired state:
SOC sees retrieval, prompt risk, tool parameters and policy route.
Risk / control concern:
SOC sees only normal API call.

Token misuse

Agent token used outside task scope.

Normal / desired state:
SOC correlates identity mesh violation and tool request.
Risk / control concern:
Appears as normal service-account traffic.
Failure scenarios

Semantic attack missed

SOC lacks AI-specific detection.

Gateway bypass

Agent calls API directly.

Grey failure

Agent degrades decision quality without clear breach.
