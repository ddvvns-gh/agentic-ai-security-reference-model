Govern external agents interacting with enterprise data, APIs or workflows.

External and partner agents operate outside enterprise ownership but may interact with enterprise systems. This capability constrains those interactions through federation, contracts, API scopes, provenance and monitoring.

Purpose and strategic objective

Purpose

To enable controlled collaboration with external AI agents without inheriting unmanaged trust.

Strategic objective

Protect enterprise APIs, data and workflows from untrusted or weakly governed external agents.

Architectural placement

Why this sits here

This sits in Layer 0 because partner agents are external actors at the AI attack surface.

Partner agents are not inside the enterprise trust boundary by default.
Problem statement

External agents may have unknown models, policies, memory, logging, training rules and security posture.

Capability description

What the capability does

  • Defines external-agent onboarding requirements.
  • Applies API scopes, rate limits and data boundaries.
  • Requires contractual and technical trust controls.
  • Monitors partner-agent activity.
  • Maintains evidence for partner interactions.

What the capability does not do

  • It does not make external agents enterprise-controlled.
  • It does not replace third-party risk management.
  • It does not expose internal data without policy approval.

Enterprise examples

Supplier support agent

A supplier agent updates service tickets.

Normal / desired state:
It uses scoped APIs and audited interactions.
Risk / control concern:
It reads unrelated operational data.

Partner analytics agent

Partner agent requests shared data extracts.

Normal / desired state:
Approved fields and retention rules apply.
Risk / control concern:
Partner stores sensitive data outside agreement.
Failure scenarios

Unknown partner model

External agent uses an unassessed foundation model.

Overbroad API token

Partner agent receives broad API access.

No evidence sharing

Enterprise cannot obtain partner logs after incident.
