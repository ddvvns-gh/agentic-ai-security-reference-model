C-TCC-002

Token & Session Revocation

On containment, revoke every credential, token, delegated authority and session the agent holds — completely and verifiably — so it cannot continue to act or be re-used.

Control IDC-TCC-002
TypeCorrective
Trust domainIdentity
Enforcement pointIdentity / secrets plane
Failure postureFail-closed
Parent capabilityBox 34 · Terminate, Contain & Compensate — Hard
Control objective
Close the access path of a contained agent. Killing the runtime is not enough if the agent's credentials remain valid; this control ensures the agent's identity can no longer authenticate or authorise any action, including through delegated child agents.
Control statement
On containment, all of the agent's active credentials, tokens, delegated authorities and sessions must be revoked completely and verifiably. Partial revocation must be detected and remediated.
Design & operation
  1. Enumerate the agent's identity, active tokens, sessions and any delegated child-agent grants (Boxes 22, 25).
  2. Revoke all of them across IAM, secrets and the gateway.
  3. Verify no valid credential or session remains (completeness check).
  4. Record the revocation set and completeness (E-TCC-003).
  5. Ephemeral, short-lived identity (Box 25) bounds the residual window.
Inputs / outputs

Inputs

  • Agent identity + delegation lineage
  • Active token/session inventory
  • Child-agent grant tree

Outputs

  • Revocation actions executed
  • Completeness result (M-TCC-002)
  • Revocation Record (E-TCC-003)

Test method / assurance

Design effectiveness

  • Confirm enumeration covers delegated child-agent grants and out-of-band keys.
  • Confirm a post-revocation validity check exists.
  • Confirm reliance on short-lived identity to bound exposure.

Operating effectiveness

  • Drill: contain an agent and confirm no credential/session authenticates afterwards.
  • Measure revocation completeness (M-TCC-002 = 100%).
  • Test child-agent grant cascade revocation.
Failure modes & residual risk
  • Cached / long-lived tokens. Credentials valid until expiry evade instant revocation; mitigated by short TTLs.
  • Out-of-band API keys. Keys outside IAM may be missed — the dominant residual.
  • Residual. Low–Medium, lowest where identity is fully ephemeral (Box 25).
Regulatory mapping
  • APRA CPS 234 — timely access revocation.
  • Zero-trust for agents (CSA Agentic Trust Framework).
  • ASD/CISA — least privilege and revocation.
Implementation notes (vendor-neutral)
Integrate with enterprise IAM/secrets and the agent identity plane (Box 25); revoke the full delegation lineage, not just the leaf identity. Prefer ephemeral credentials so revocation is fast by construction.
