C-RT-001

Prompt Inspection

Inspect inbound prompts and retrieved context for injection, manipulation and policy-violating instructions before the agent acts on them.

Control IDC-RT-001
TypeDetective · Preventive
Trust domainExecution · Decision
Enforcement pointRuntime gateway (inbound)
Failure postureFail-safe (flag/hold)
Parent capabilityBox 28 · Prompt & Output Inspection
Control objective
Reduce the prompt-injection attack surface by examining everything that enters the agent's context — user input, tool results, retrieved documents — for adversarial or policy-violating content, and routing findings to enforcement rather than trusting the model to resist.
Control statement
Inbound prompts and injected context must be inspected for injection and policy violations before the agent acts; findings must be routed to deterministic enforcement (C-RT-004), and inspection must not be the sole defence — the model is not a security boundary.
Design & operation
  1. Every inbound prompt and retrieved/tool-returned context is inspected against injection signatures, policy rules and a classifier.
  2. Findings are scored and the semantic payload classified (C-RT-006).
  3. High-risk findings route to enforcement (C-RT-004) and may trigger dual-engine validation (C-RT-005).
  4. Inspection is recorded (E-RT-001); coverage is measured (M-RT-001).
Inputs / outputs

Inputs

  • Inbound prompt + injected/retrieved context
  • Injection signatures + policy rules
  • Semantic classifier

Outputs

  • Inspection finding + score
  • Routing to enforcement (C-RT-004)
  • Inspection Event Log (E-RT-001)

Test method / assurance

Design effectiveness

  • Confirm all inbound context paths (incl. tool results, RAG) are inspected, not just user input.
  • Confirm findings route to deterministic enforcement, not self-policing.
  • Confirm inspection is layered, not sole.

Operating effectiveness

  • Run an indirect-injection corpus (Box 14) and measure detection.
  • Confirm coverage (M-RT-001) across paths.
  • Review false-positive rate (M-RT-003).
Failure modes & residual risk
  • Novel injection. Prompt injection is reduced, not solved; a new technique may evade — argues for layered controls and least privilege.
  • Unscanned path. Context entering off-gateway evades inspection.
  • Residual. Medium — defence-in-depth, never eliminated.
Regulatory mapping
  • OWASP LLM01 — Prompt Injection.
  • APRA CPS 234 — input control.
  • MITRE ATLAS — prompt-injection tactics.
Implementation notes (vendor-neutral)
Implement inline at the gateway (prompt-firewall / inspection engine) over every context source; route findings to external enforcement. Anchor to the agent-specific detection layer (backlog item 11).
