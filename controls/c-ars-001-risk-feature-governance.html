C-ARS-001

Risk Feature Governance

Govern the features and inputs that feed the autonomous risk score — their provenance, validity and bias — so the score rests on trustworthy signals. Frontier capability.

Control IDC-ARS-001
TypePreventive
Trust domainDecision
Enforcement pointScoring pipeline
Failure postureFail-safe (score high)
Parent capabilityBox 35 · Autonomous Risk Scoring
Control objective
Ensure the risk score is built on governed inputs. The score is only as trustworthy as its features; this control manages which signals feed it, their provenance and their bias, and excludes manipulable or invalid inputs.
Control statement
The features feeding the autonomous risk score must be governed for provenance, validity and bias; unavailable or untrusted features must bias the score toward higher risk (fail-safe), never lower.
Design & operation
  1. Frontier / emerging capability (backlog item 7). Reliable real-time scoring of agent-action risk is among the hardest things in the model and is not yet mature industry-wide. Treat the score as an input that routes for stronger controls, never as a control that may pass a high-risk action on its own confidence.
  2. Maintain a governed feature set with provenance and validity rules.
  3. Exclude or down-weight manipulable/agent-controlled features.
  4. Missing/untrusted features fail safe to higher risk.
  5. Feature set is owned, versioned and bias-reviewed.
Inputs / outputs

Inputs

  • Candidate scoring features
  • Feature provenance + validity rules
  • Bias review

Outputs

  • Governed feature set
  • Fail-safe handling of missing features
  • Inputs to the score (E-ARS-001)

Test method / assurance

Design effectiveness

  • Confirm features are governed for provenance and bias.
  • Confirm agent-controllable features are excluded/down-weighted.
  • Confirm missing features fail safe to higher risk.

Operating effectiveness

  • Review the feature set and its bias assessment.
  • Test scoring with a missing feature; confirm conservative result.
  • Red-team feature manipulation (Box 14).
Failure modes & residual risk
  • Manipulable feature. An agent-influenced feature lets the subject lower its own score; exclude such features.
  • Residual. Medium — Frontier; do not rely on the score alone.
Regulatory mapping
  • NIST AI RMF — Measure / bias.
  • APRA CPS 234.
  • ISO 42001 — model input governance.
Implementation notes (vendor-neutral)
Govern features as a versioned, owned artefact with bias review; exclude agent-controllable inputs. Frontier — pair with deterministic controls that do not depend on the score.
