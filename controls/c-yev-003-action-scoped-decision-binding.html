C-YEV-003

Action-Scoped Decision Binding

Bind every soft decision to the specific action only — so allowing one action can never widen a policy boundary or set a precedent.

Control IDC-YEV-003
TypePreventive
Trust domainDecision
Enforcement pointDecision-routing gateway
Failure postureFail-closed
Parent capabilityBox 37 · Yield & Evaluate — Soft
Control objective
Keep the soft path from quietly changing the rules. A decision to allow or deny applies to this action and no other; any change to a policy boundary requires an approved policy update and redeployment.
Control statement
A soft decision must apply only to the specific action evaluated; changing a policy boundary must require an approved policy update and redeployment, never a soft allow.
Design & operation
  1. Scope each decision to the single action and its context.
  2. Block any path by which a soft allow generalises into a standing rule change.
  3. Boundary changes route to policy governance (Box 16), not the soft path.
  4. Decision scope recorded (E-YEV-001).
Inputs / outputs

Inputs

  • Soft decision + action context
  • Policy boundary definitions

Outputs

  • Action-scoped decision
  • Boundary-change requests routed to policy governance
  • Scope recorded (E-YEV-001)

Test method / assurance

Design effectiveness

  • Confirm decisions are scoped to the single action.
  • Confirm soft allows cannot become standing rule changes.
  • Confirm boundary changes route to policy governance.

Operating effectiveness

  • Test that a soft allow does not persist as policy.
  • Confirm boundary-change attempts route to Box 16.
  • Sample decisions for action-scoping.
Failure modes & residual risk
  • Implicit precedent. Repeated soft allows treated informally as policy; mitigated by routing real changes to governance.
  • Residual. Low.
Regulatory mapping
  • NIST AI RMF — Govern.
  • APRA CPS 234.
  • Policy-as-Code Standard.
Implementation notes (vendor-neutral)
Enforce action-scoping in the gateway; route genuine boundary changes to policy-as-code governance (Box 16). The soft path resolves actions, it does not legislate.
