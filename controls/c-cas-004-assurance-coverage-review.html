C-CAS-004

Assurance Coverage Review

Periodically review which controls are actually under assurance — so the set of controls being tested keeps pace with the controls that exist.

Control IDC-CAS-004
TypeDetective
Trust domainDecision
Enforcement pointAssurance plane (periodic)
Failure postureFlag gaps
Parent capabilityBox 38 · Continuous Assurance
Control objective
Prevent assurance from quietly narrowing. As controls are added or change, the coverage review confirms they are mapped and tested, closing the blind spots where un-assured controls accumulate.
Control statement
The set of controls under continuous assurance must be reviewed periodically against the full control library; controls not mapped or not tested must be identified and prioritised for coverage.
Design & operation
  1. Reconcile the control library against the control→evidence mapping and the automated test set.
  2. Identify controls that are unmapped or untested (blind spots).
  3. Prioritise closing high-risk coverage gaps; report coverage (E-CAS-003).
  4. Couples to the reference-model expansion backlog (Box 51).
Inputs / outputs

Inputs

  • Mapping (C-CAS-001) + test set (C-CAS-002)

Outputs

  • Coverage gaps identified
  • Prioritised coverage backlog
  • Coverage Report (E-CAS-003)

Design effectiveness

  • Confirm coverage review reconciles library vs mapping vs tests.
  • Confirm gaps are prioritised, not just listed.
  • Confirm cadence.

Operating effectiveness

  • Measure continuous test coverage (M-CAS-001) trend.
  • Confirm new controls enter assurance.
  • Track coverage-gap closure.
Failure modes & residual risk
  • Lagging review. Coverage drifts between reviews; mitigated by frequent, automated reconciliation.
  • Residual. Low–Medium.
