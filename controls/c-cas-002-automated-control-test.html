C-CAS-002

Automated Control Test

Continuously and automatically test that controls operate — against their mapped evidence — rather than relying on periodic manual review.

Control IDC-CAS-002
TypeDetective
Trust domainDecision
Enforcement pointAssurance plane (continuous)
Failure postureAlert
Parent capabilityBox 38 · Continuous Assurance
Control objective
Replace point-in-time control testing with continuous automated assurance: regularly verify each control's evidence shows it operating, so failures are caught in days, not at the next audit.
Control statement
In-scope controls must be tested automatically and continuously against their mapped evidence; failed tests must raise findings and feed remediation (M-CAS-004).
Design & operation
  1. For each mapped control, run automated tests checking its evidence demonstrates operation.
  2. Failures raise control-test findings (E-CAS-001) and open remediation.
  3. Coverage of controls under continuous test measured (M-CAS-001); failures (M-CAS-003).
  4. Couples to control-effectiveness monitoring (Box 48).
Inputs / outputs

Inputs

  • Control→evidence mapping (C-CAS-001)
  • Evidence streams
  • Test definitions

Outputs

  • Continuous control test results
  • Findings + remediation triggers
  • Control Test Result (E-CAS-001)

Test method / assurance

Design effectiveness

  • Confirm automated tests exist for mapped controls.
  • Confirm failures raise findings + remediation.
  • Confirm test definitions are owned/versioned.

Operating effectiveness

  • Measure continuous test coverage (M-CAS-001).
  • Confirm a seeded control failure is caught.
  • Track failed control tests (M-CAS-003) and MTTR (M-CAS-004).
Failure modes & residual risk
  • Test gap. A control with no automated test relies on slower review; prioritise via coverage review.
  • Residual. Low–Medium.
Regulatory mapping
  • APRA CPS 234 — operating-effectiveness testing.
  • NIST AI RMF — Measure.
  • Continuous controls monitoring practice.
Implementation notes (vendor-neutral)
Run as continuous control monitoring over the evidence streams (Box 46); couple to control-effectiveness monitoring (Box 48). This is what makes assurance continuous rather than periodic.
