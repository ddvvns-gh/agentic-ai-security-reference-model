C-ESC-003

Approval Evidence Package

Assemble a complete, accurate decision context for the approver — what the action is, why it was escalated, its risk and reversibility — so the approval is informed.

Control IDC-ESC-003
TypeEvidentiary
Trust domainDecision
Enforcement pointEscalation workflow
Failure postureRecord-or-hold
Parent capabilityBox 39 · Pause & Escalate — High Impact
Control objective
Make high-impact approvals genuine decisions rather than rubber stamps. The quality of an approval is bounded by the quality of the context the approver sees; this control standardises that context.
Control statement
Every escalated action must be accompanied by a standardised evidence package giving the approver the action detail, escalation reason, risk/score, reversibility and relevant lineage; an action whose package cannot be assembled must remain held.
Design & operation
  1. On pause, gather the action parameters, the escalation reason (threshold/score), risk and reversibility, and the relevant decision lineage (Box 43 / Box 33).
  2. Render a consistent package for the approver, including the recommended options.
  3. Persist the package as escalation evidence (E-ESC-001).
  4. Incompleteness holds the action rather than presenting a partial picture.
Inputs / outputs

Inputs

  • Paused action + escalation reason
  • Risk/score + reversibility
  • Decision lineage references

Outputs

  • Standardised approval package (E-ESC-001)
  • Hold if package incomplete

Test method / assurance

Design effectiveness

  • Confirm the package schema covers action, reason, risk, reversibility and lineage.
  • Confirm incomplete packages hold rather than present partial context.
  • Confirm options/recommendation are included.

Operating effectiveness

  • Review sampled packages for completeness and accuracy.
  • Correlate package quality with approval-overturn / decision-time.
  • Test the hold path on a deliberately incomplete case.
Failure modes & residual risk
  • Misleading context. An inaccurate package produces a bad-but-confident approval; mitigated by sourcing from authoritative lineage.
  • Residual. Low–Medium, bounded by lineage quality (Box 43).
Regulatory mapping
  • APRA CPS 230 — informed accountable decisions.
  • EU AI Act — human oversight with adequate information.
  • ISO 42001 — decision support.
Implementation notes (vendor-neutral)
Assemble from authoritative sources (commitment register, decision lineage, risk score); never let the requesting agent compose its own justification. Standardise the schema so approvers build pattern familiarity.
