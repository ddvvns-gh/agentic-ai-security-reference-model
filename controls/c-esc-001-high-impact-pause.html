C-ESC-001

High-Impact Pause

Pause a high-impact action before it executes and hold it until an accountable human has decided — the graceful, legitimate counterpart to the hard kill path.

Control IDC-ESC-001
TypePreventive
Trust domainDecision
Enforcement pointDecision-routing gateway (pre-execution)
Failure postureFail-closed
Parent capabilityBox 39 · Pause & Escalate — High Impact
Control objective
Ensure that legitimately high-impact actions (material, irreversible or customer-affecting) cannot execute on agent autonomy alone. Unlike the hard response (Box 34, for compromise), this is the planned escalation route for actions that are simply too consequential to auto-proceed.
Control statement
An action classified high-impact (by threshold or risk score) must be paused before execution and held pending an accountable human decision. The pause must be enforced outside the agent and must fail closed if no decision is reached within the SLA.
Design & operation
  1. A high-impact classification arrives — from threshold evaluation (Box 33 C-OCC-008), autonomous risk scoring (Box 35) or policy.
  2. The decision-routing gateway pauses the action and places it in an escalation queue.
  3. An approval evidence package (C-ESC-003) is assembled and routed to the accountable approver (C-ESC-002).
  4. On approve → execute under a state lock; on reject or SLA timeout → fail closed.
  5. All timings and outcomes are recorded for coverage and decision-time metrics.
Inputs / outputs

Inputs

  • High-impact classification (threshold / risk score)
  • Action context
  • Escalation SLA

Outputs

  • Paused action in escalation queue
  • Trigger to assemble package (C-ESC-003) and route (C-ESC-002)
  • Execute / reject / timeout outcome

Test method / assurance

Design effectiveness

  • Confirm all high-impact classifiers route into the pause.
  • Confirm the pause is enforced outside the agent and fails closed on timeout.
  • Confirm execution after approval is state-locked (Box 33 C-OCC-003).

Operating effectiveness

  • Drill a high-impact action; confirm it pauses and cannot execute without a decision.
  • Measure approval coverage (M-ESC-001) and decision time (M-ESC-002).
  • Test SLA timeout fails closed.
Failure modes & residual risk
  • Classification miss. A high-impact action not classed as such bypasses the pause — residual inherited from threshold/scoring coverage.
  • SLA pressure. Aggressive timeouts that fail open would defeat the control; must fail closed.
  • Residual. Low with fail-closed pause and good classification.
Regulatory mapping
  • APRA CPS 230 — human control over material automated actions.
  • APRA CPS 234 — control design/operating evidence.
  • ASD/CISA — graduated autonomy; human controllers for high-impact.
Implementation notes (vendor-neutral)
Implement in the decision-routing gateway as a fail-closed hold with an escalation queue; reuse the commitment state-lock (Box 33) for post-approval execution. Distinct from Box 34: this path assumes a legitimate-but-consequential action, not a compromise.
