C-SBOX-005

Micro-Isolation Profile

Apply a tighter micro-isolation profile to high-risk dynamic execution — stronger isolation, narrower capabilities, no network — than the standard sandbox.

Control IDC-SBOX-005
TypePreventive
Trust domainExecution
Enforcement pointSandbox runtime (high-risk)
Failure postureFail-closed
Parent capabilityBox 27 · Ephemeral Sandboxing for Dynamic Execution
Control objective
Match isolation strength to risk. High-risk execution (untrusted code, sensitive context) gets a hardened micro-isolation profile beyond the baseline, minimising what even a successful exploit could reach.
Control statement
High-risk dynamic execution must run under a hardened micro-isolation profile (e.g. no network, minimal syscalls, microVM-grade isolation); high-risk execution must not run under only the baseline profile.
Design & operation
  1. Classify execution risk; high-risk routes to a micro-isolation profile.
  2. The profile tightens isolation: no/whitelisted network, minimal syscalls, strongest isolation tech.
  3. Coverage of high-risk execution under micro-isolation measured (M-SBOX-004).
  4. Profile attested (E-SBOX-003).
Inputs / outputs

Inputs

  • Execution risk classification
  • Micro-isolation profile

Outputs

  • Hardened high-risk sandbox
  • Micro-isolation coverage (M-SBOX-004)
  • Isolation attestation (E-SBOX-003)

Test method / assurance

Design effectiveness

  • Confirm high-risk execution is classified and routed to micro-isolation.
  • Confirm the profile is materially stronger than baseline.
  • Confirm coverage.

Operating effectiveness

  • Confirm high-risk execution gets no/whitelisted network and minimal syscalls.
  • Measure high-risk micro-isolation coverage (M-SBOX-004).
  • Red-team escape from the micro-isolation profile (Box 14).
Failure modes & residual risk
  • Mis-classification. High-risk execution under baseline isolation; fail-safe to stronger profile.
  • Residual. Low–Medium.
Regulatory mapping
  • OWASP Agentic — ASI05.
  • APRA CPS 234.
  • Defence-in-depth.
Implementation notes (vendor-neutral)
Provide a hardened micro-isolation tier (microVM, no network, minimal syscalls) for high-risk execution; route by classification and attest.
