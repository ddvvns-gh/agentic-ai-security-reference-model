C-MEM-001

Memory Classification

Classify every memory entry by sensitivity, type and trust at write time — so recall, retention and sanitisation can act on what the memory actually is.

Control IDC-MEM-001
TypePreventive
Trust domainDecision
Enforcement pointMemory plane (write)
Failure postureFail-safe (treat as sensitive/untrusted)
Parent capabilityBox 32 · Agent Memory Governance
Control objective
Give the memory controls a meaning-level signal. Classifying memory (sensitive vs not, fact vs instruction, trusted vs untrusted source) is the precondition for recall gating, retention and poisoning defence.
Control statement
Every memory write must be classified for sensitivity, type and trust; unclassifiable entries must be treated as sensitive and untrusted.
Design & operation
  1. Classify each write along sensitivity, content-type (fact/instruction/preference) and source-trust dimensions.
  2. Classification drives recall gating (C-MEM-005), retention/epoch (C-MEM-003) and sanitisation (C-MEM-004).
  3. Ambiguous entries default to the most protective handling.
  4. Classification stored with the entry and its provenance (E-MEM-001).
Inputs / outputs

Inputs

  • Memory write + content
  • Source/provenance
  • Classification taxonomy

Outputs

  • Classified memory entry
  • Inputs to recall/retention/sanitisation
  • Provenance record (E-MEM-001)

Test method / assurance

Design effectiveness

  • Confirm classification at write covers sensitivity, type and trust.
  • Confirm ambiguous entries fail safe.
  • Confirm classification drives downstream controls.

Operating effectiveness

  • Sample memory entries for correct classification.
  • Confirm sensitive content is tagged and handled.
  • Review misclassification feedback.
Failure modes & residual risk
  • Misclassification. Untrusted content tagged trusted weakens recall gating; mitigated by fail-safe defaults.
  • Residual. Low–Medium.
Regulatory mapping
  • Privacy Act — data classification/retention.
  • OWASP Agentic — memory poisoning.
  • Memory Epoch Policy / Data Classification Standard.
Implementation notes (vendor-neutral)
Classify at the memory-write boundary; bind to the Memory Epoch Policy and Data Classification Standard. Classification is the foundation the other memory controls stand on.
