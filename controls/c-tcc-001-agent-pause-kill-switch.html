C-TCC-001

Agent Pause / Kill-Switch

An immediate, authoritative means to pause or terminate any in-scope agent and halt its in-flight actions — so harm stops at the moment of decision, not after the workflow completes.

Control IDC-TCC-001
TypeCorrective · Preventive
Trust domainExecution
Enforcement pointControl plane (out-of-band, immediate)
Failure postureFail-safe (halt)
Parent capabilityBox 34 · Terminate, Contain & Compensate — Hard
Control objective
Guarantee the enterprise can stop a misbehaving or compromised agent at once. The control halts the agent's reasoning loop and cancels in-flight actions out-of-band, so a runaway or hijacked agent cannot continue acting while a response is organised.
Control statement
A single authoritative control must be able to immediately pause or terminate any in-scope agent and halt its in-flight actions. The kill/pause must act out-of-band of the agent — the agent cannot veto or evade it — and must be recorded.
Design & operation
  1. A containment trigger arrives — manually from the SOC, or automatically from autonomous risk scoring (Box 35), monitoring (Box 33 C-OCC-006) or threat detection/IR (Box 47).
  2. The control plane issues an out-of-band halt to the agent runtime/orchestrator and moves the agent to a contained state.
  3. In-flight tool calls are cancelled or blocked, coordinated with Tool & Egress Block (C-TCC-003).
  4. Handoff to revocation (C-TCC-002) and memory quarantine (C-TCC-004).
  5. A kill/pause record (E-TCC-002) is written and a residual-activity check is started.
Inputs / outputs

Inputs

  • Containment trigger + severity
  • Agent runtime handle (pre-provisioned)
  • Blast-radius scope (single agent / class)

Outputs

  • Agent halted; contained state
  • Kill/Pause Record (E-TCC-002)
  • Handoff to C-TCC-002 / C-TCC-003 / C-TCC-004

Test method / assurance

Design effectiveness

  • Confirm a pre-provisioned, out-of-band kill handle exists for every in-scope agent and cannot be evaded by the agent.
  • Confirm all trigger sources (manual + automated) are wired.
  • Confirm in-flight action cancellation is coordinated with the gateway.

Operating effectiveness

  • Run an unannounced kill drill; measure mean time to containment (M-TCC-001).
  • Confirm zero agent actions after the kill is issued (M-TCC-003 = 0).
  • Confirm a complete kill/pause record is produced.
Failure modes & residual risk
  • Unreachable runtime. Agents on external/un-instrumented runtimes may not be killable out-of-band — a coverage gap.
  • Already-sent irreversible action. A kill cannot un-send; residual harm passes to compensation (C-TCC-005).
  • Residual. Low with full runtime coverage; Medium where coverage is partial.
Regulatory mapping
  • APRA CPS 230 — ability to contain operational disruption.
  • APRA CPS 234 — incident response capability.
  • ASD/CISA (Apr 2026) — contain compromised agents; treat capable agents as untrusted.
Implementation notes (vendor-neutral)
Implement as an orchestrator-level kill API plus gateway action-block, with per-agent kill handles pre-provisioned and regularly tested; drive from SOAR. The control is defined by the out-of-band, un-vetoable halt — not a product.
