M-OCC-001

Unauthorized Commitment Attempts

The volume of commitment attempts blocked because the acting agent lacked sufficient delegated authority — a leading indicator of attack pressure, reasoning error or authority misconfiguration.

Metric IDM-OCC-001
TypeSecurity · leading
SignalInvestigate on spike
CadenceContinuous; reviewed monthly
OwnerSecurity / Risk
Parent capabilityBox 33 · Output Commitment Control
Definition
The count of commitment attempts denied by Commitment Authority Validation (C-OCC-001) because the acting agent — or a principal in its delegation chain — did not hold sufficient authority for the action type, value band or purpose. Each event represents a commitment the control successfully prevented; the metric measures the pressure on the boundary, not control failure.
Calculation
M-OCC-001(period) = COUNT( authority-validation decisions WHERE outcome = DENY(insufficient_authority) )

Normalised rate = ( denied_for_authority / total_commitment_attempts ) × 1,000
Report both the absolute count (operational) and the normalised rate per 1,000 commitments (trendable across changing volumes). Segment by agent, declared purpose, counterparty class and value band to localise spikes.
Target / signal
No fixed target — this is a monitor-and-investigate signal, not a pass/fail threshold. Establish a rolling baseline; a sustained rise or a sharp spike is the trigger. A non-zero floor is expected and healthy (the control is doing its job). Zero over a long window is itself suspect — it usually means the control is not actually in the commitment path, not that no attempts occur.
Data source & collection

Source

  • Authority Validation Records (E-OCC-001) with outcome = deny(authority)
  • Commitment gateway decision logs

Collection point

  • Control-plane / commitment-gateway telemetry
  • Aggregated via structured agentic telemetry (Box 46)
  • Correlation key: agent ID + commitment ID
Interpretation & response
  • Spike on one agent/purpose → likely prompt-injection campaign or a broken upstream prompting out-of-authority actions; investigate the agent and recent changes.
  • Broad rise across agents → often an authority-matrix change (over-tightening) or a mis-scoped Prodspec; reconcile with the Delegated Authority Framework version history.
  • Drop to zero → verify C-OCC-001 is still in the commitment path and classification (C-OCC-002) is still flagging commitments.
