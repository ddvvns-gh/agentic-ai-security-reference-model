R-DATA-006

Vector Store Leakage

Sensitive data embedded in a vector store leaks — through over-broad retrieval, failed deletion, or weak access/residency controls.

Risk IDR-DATA-006
CategoryData / Confidentiality
Trust domainExecution
ReversibilityIrreversible (disclosure)
OwnerSecurity · Privacy
Parent capabilityBox 29 · Data Boundary & RAG Governance
Description
Vector stores are an easily-overlooked copy of sensitive source data. Without classification, access control, residency and reliable deletion, they leak the very data the source systems protect — including persisting data that should have been erased.
Threat scenario
An erasure request is honoured in the source system but the corresponding embeddings remain in the vector store and continue to surface the 'deleted' personal data in retrievals.
Assessment
LikelihoodMediumVector stores are frequently under-governed.
ImpactHighPersistent sensitive-data exposure; erasure failure.
InherentHighBefore controls.
